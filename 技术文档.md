# AI 短视频自动生成系统 — 技术文档

> 版本：V1.0
> 日期：2026年7月10日
> 用途：素材扫描 → AI 识图 → AI 脚本 → TTS 配音 → FFmpeg 合成 → 输出短视频

---

## 一、项目概述

全自动短视频生成流水线。支持抖音/小红书/视频号三个平台的差异化参数。AI 视觉识别素材内容 → AI 生成分镜脚本 → edge-tts 配音 → FFmpeg 合成（字幕+模糊背景+画质增强+BGM）→ 输出成品视频。

> ⚠️ 本项目的练习意义大于实际意义。AI 生成的内容无法替代真人拍摄的真实感和互动性。

---

## 二、技术栈

| 层面 | 技术 | 说明 |
|------|------|------|
| AI 脚本 | DeepSeek API / 千问 API | 双模型自动切换，失败降级默认脚本 |
| 视觉识别 | 千问 VL（Qwen-VL-Plus） | 识别素材画面内容 |
| TTS 配音 | edge-tts | Microsoft 免费 TTS，多音色 |
| 视频合成 | FFmpeg（imageio-ffmpeg） | 拼接/字幕/BGM/滤镜 |
| 字幕渲染 | Pillow（PIL） | 渲染 PNG 字幕图片 |
| 平台参数 | 三平台差异化配置 | 抖音/小红书/视频号 |

---

## 三、系统架构

```
main.py（生成视频入口）
  │
  ├─ config.py      → FFmpeg路径 + API密钥 + 视频参数
  ├─ platforms.py   → 三平台配置（尺寸/音色/字幕样式）
  ├─ materials.py   → 扫描素材 + AI识图（Qwen VL）
  ├─ script_gen.py  → AI脚本生成（DeepSeek/Qwen）
  ├─ voice.py       → edge-tts 逐句配音
  ├─ scenes.py      → 字幕PNG + 单镜头FFmpeg合成
  └─ compose.py     → 镜头拼接 + BGM叠加

数据流：
  素材文件夹 → 扫描 → AI识图 → AI脚本 → 配音 → 单镜头 → 拼接+BGM → output.mp4
```

---

## 四、模块说明

### 4.1 素材识别（materials.py）

扫描文件夹 → 图片直接送 AI → 视频抽帧再送 AI → Qwen VL 返回一句话画面描述 → 传给脚本生成器做精准编排

### 4.2 脚本生成（script_gen.py）

`_pick_ai_provider()` 自动选择可用模型：DeepSeek > Qwen > 默认模板。AI 不可用时降级为预设文案，不中断流程。

### 4.3 视频合成（scenes.py + compose.py）

滤镜链：原始素材 → 模糊背景 + 锐化前景 → 叠加字幕 PNG → 混入配音 → 单镜头 → 拼接 → 叠加 BGM

### 4.4 平台适配（platforms.py）

| 平台 | 分辨率 | 每镜时长 | 风格 |
|------|--------|:--:|------|
| 抖音 | 1080×1920 | 1.5-3s | 快节奏、冲击力 |
| 小红书 | 1080×1440 | 2.5-4s | 精致种草 |
| 视频号 | 1080×1920 | 3-5s | 稳重专业 |

## 五、使用方式

```bash
python main.py          # 交互选平台
python main.py 抖音      # 直接指定平台
```

## 六、开发踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| run_ffmpeg 散落各处 | voice/materials/scenes/compose 都需要 | 统一放 config.py，避免循环导入 |
| 字幕中文乱码 | FFmpeg drawtext 中文字体兼容差 | 改用 Pillow 渲染 PNG，FFmpeg 叠加图片 |
| AI 脚本返回非 JSON | DeepSeek 有时包在 ```json``` 里 | 解析前 strip markdown 代码块标记 |
